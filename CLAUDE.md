# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

STM32H743IIK firmware (CubeMX-generated base) that receives files over USB CDC ACM and writes them to an SD card, while driving an ST7735 LCD via LVGL. The PC sends files with `tools/pc/send_file.py`; the device displays status/animation on the LCD. Built with Keil MDK-ARM using the **AC6 (ArmClang)** toolchain.

The project is a git repo (`origin` = github.com/yjrqz777/h7rec). Keep edits narrow and avoid broad generated-file churn.

## Build And Tooling

- Open `MDK-ARM/h7rec.uvprojx` in Keil uVision and build. Target device is `STM32H743IIKx`, toolchain AC6 (`<uAC6>1</uAC6>`).
- Build output goes to `MDK-ARM/h7rec/` — treat as generated; the hex lands there. `.gitignore` excludes Keil build products, logs, lock files, user/workspace files.
- Preprocessor defines (project-level): `USE_PWR_LDO_SUPPLY, USE_HAL_DRIVER, STM32H743xx, LV_CONF_INCLUDE_SIMPLE, LV_LVGL_H_INCLUDE_SIMPLE`.
- `moveset.bat` copies `MDK-ARM/.vscode` into the root `.vscode`. There are two `c_cpp_properties.json` copies (root + `MDK-ARM/.vscode`); the root one is generated by `moveset.bat`, so avoid editing only one unless divergence is intentional.

### AC6 / FreeRTOS portable caveat (important)

CubeMX regenerates FreeRTOS with the **AC5** RVDS portable, which does not build under AC6. After any CubeMX regeneration you must re-apply the AC6 portable:

- Copy `External/FreeRTOS-Kernel-10.3.1-kernel-only/portable/GCC/ARM_CM4F` over `Middlewares/Third_Party/FreeRTOS/Source/portable/RVDS/ARM_CM4F`.

## Application Architecture

The key design split: **CubeMX-generated `Core/` files stay thin and own only RTOS/peripheral setup. All application logic lives under `App/`.** `freertos.c` creates `defaultTask` and then calls `AppRuntime_CreateTasks()` from its USER CODE region; `StartDefaultTask` delegates to `AppRuntime_DefaultTask()`.

### Boot flow

`main()` (`Core/Src/main.c`): SEGGER RTT init → MPU config → HAL init → clock → GPIO/RTC/SPI2/TIM2/SDMMC1/FATFS init → start TIM2 PWM (LCD backlight) → `osKernelInitialize()` → `MX_FREERTOS_Init()` → `osKernelStart()`. Note `MX_USB_OTG_FS_PCD_Init()` is commented out — USB is driven by CherryUSB, not HAL PCD.

### Task topology (`App/Runtime/app_runtime.c`)

`AppRuntime_CreateTasks()` initializes modules then spawns:
- **defaultTask** → `AppRuntime_DefaultTask`: runs `LCD_Test()` then idles (the periodic RTC/SD/RX status-refresh loop is currently commented out).
- **sdManagerTask** → `SdManager_Task`: background SD mount/retry.
- **fileRxTask** → `FileRx_Task`: USB CDC file-receive protocol + SD write.
- **uiTask** → `AppGui_Task`: LVGL init + `lv_task_handler()` loop.
- **usbTask** → `StartUsbTask`: one-shot, calls `CherryUSB_DeviceInit()` after a short delay then exits. Gated by `CHERRYUSB_AUTO_START`.

### Modules

- `App/FileTransfer/file_rx.{c,h}` — USB CDC file receiver. Protocol: `PUT → READY → [BLK+payload+ACK]×N → END → DONE`, with per-block CRC and whole-file CRC. Data arrives via `FileRx_OnUsbData()` (CDC OUT callback) into a 32 KB ring buffer; `FileRx_Task` parses it asynchronously and writes to `0:/rx`. Tunables (chunk size, paths, debug dump) are macros at the top of `file_rx.h`.
- `App/Storage/sd_manager.{c,h}` — centralizes SDMMC/FatFs mount/unmount/capacity behind a state machine (`SdManagerState`). All SD access should go through here; other tasks query status rather than touching the driver. Mount drive is `0:`.
- `App/Gui/` — `app_gui.c` (LVGL UI task; currently shows a zooming GIF from `P:/rx/miao.gif`) plus `LvglPort/` display/input/filesystem porting layers and `lv_conf.h`. LVGL FS driver letter is `P:` and bridges to FatFs (`ff.h`).
- `BSP/CherryUSB_port/` — CherryUSB CDC ACM device app (`cherryusb_app.c`), DWC2 port, and `usb_config.h`. Provides `CherryUSB_DeviceInit/CdcSend/CdcCanSend`.

### LCD timebase coupling

`vApplicationTickHook()` in `freertos.c` calls `lv_tick_inc(1)` every FreeRTOS tick — LVGL's timebase is the RTOS tick. HAL's timebase is separate: TIM17 drives `HAL_IncTick()` via `HAL_TIM_PeriodElapsedCallback` (`stm32h7xx_hal_timebase_tim.c`), so do not assume SysTick owns the HAL time base.

## Layout

- `Core/Inc`, `Core/Src` — CubeMX-generated app, peripheral, ISR, memory-map, and RTOS glue.
- `App/` — application layer (Runtime, FileTransfer, Storage, Gui). Not CubeMX-managed; edit freely in normal C style.
- `BSP/ST7735` — LCD driver, fonts, logo. `BSP/CherryUSB_port` — CherryUSB STM32 DWC2 port.
- `Drivers/` — STM32H7 HAL + CMSIS. `Middlewares/Third_Party/FreeRTOS` — kernel + CMSIS-RTOS2 wrapper. `FATFS/` — FatFs app glue.
- `External/` — vendored libs: CherryUSB-1.6.1, lvgl-8.4.0, rlottie-0.2, SEGGER_RTT_V798a, FreeRTOS-Kernel-10.3.1 (source of the AC6 portable).
- `tools/pc/` — `send_file.py` PC sender (`pip install pyserial`; `python tools/pc/send_file.py COM7 file.hex`). See `tools/pc/README.md`.
- `doc/` — porting/QA notes (CherryUSB+RTT, CDC→SD transfer).

## RTOS Config (`Core/Inc/FreeRTOSConfig.h`)

Preemptive, 1 kHz tick, `configMAX_PRIORITIES` 56, `configMINIMAL_STACK_SIZE` 1024, `configTOTAL_HEAP_SIZE` 32768 bytes, `configUSE_TICK_HOOK` 1 (required for the `lv_tick_inc` hook above). Uses CMSIS-RTOS2 (`cmsis_os.h`/`cmsis_os2.h`, `osThreadNew`, `osDelay`).

## Code Generation Rules

- Most `Core/` files are CubeMX-generated. Preserve `/* USER CODE BEGIN ... */` / `/* USER CODE END ... */` regions and place manual code inside them. The hook into the app layer lives in those regions of `main.c` and `freertos.c`.
- For peripheral config changes, prefer editing the `.ioc` (`h7rec.ioc`) and regenerating, then review the diff — and remember to re-apply the AC6 FreeRTOS portable afterward.
- Don't casually reformat generated files; small behavioral edits review far better with CubeMX formatting intact.

## Peripheral Notes

- LCD pins (`Core/Inc/main.h`, `gpio.c`): DC/RS on PD13, reset PD12, CS PB12, backlight PA5/TIM2_CH1, key input PB1.
- SPI2 (`spi.c`): master, 1-line TX, 8-bit, CPOL low/CPHA first edge, software NSS, prescaler 16, SCK PB13, MOSI PB15.
- TIM2 PWM (`tim.c`): backlight, prescaler `480-1`, period `1000-1`, channel 1, low polarity. Started before the RTOS in `main.c`.
- SDMMC1 + FatFs for the SD card (drive `0:`); USB OTG FS pins driven by the CherryUSB DWC2 port, not HAL PCD.
- `BSP/ST7735/lcd.h` defines the panel variant (`TFT96`); change deliberately for a different panel.

## Editing Expectations

- Keep changes Keil/ArmClang-compatible C unless the toolchain is intentionally migrated.
- Use HAL + CMSIS-RTOS2 APIs matching existing style.
- Treat `BSP/ST7735`, `External/*`, and `BSP/CherryUSB_port` as imported/ported code: fix integration issues narrowly, avoid large cosmetic rewrites.
- When touching generated CubeMX code and BSP/middleware in the same change, separate the rationale clearly in the final response.
