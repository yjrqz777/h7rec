# Repository Guidelines

## Project Overview

- This is an STM32CubeMX-generated STM32H743 firmware project targeting Keil MDK-ARM/uVision. The main project file is `MDK-ARM/h7rec.uvprojx`.
- The workspace is not currently a git repository. Keep edits narrow and avoid broad generated-file churn.
- `Core/Src/main.c` configures the MPU, initializes HAL and the CubeMX peripherals, starts TIM2 PWM for LCD backlight, calls `LCD_Test()`, then starts CMSIS-RTOS2 with `MX_FREERTOS_Init()`.
- `Core/Src/freertos.c` creates only `defaultTask`. The task reads RTC time/date and refreshes ST7735 LCD text with the time and HAL tick; `LED_Blink()` currently contains only commented-out GPIO code.

## Layout

- `Core/Inc` and `Core/Src`: CubeMX-generated application, peripheral, interrupt, memory-map, and RTOS glue code.
- `Drivers/`: STM32H7 HAL and CMSIS headers/sources generated or copied with CubeMX.
- `Middlewares/Third_Party/FreeRTOS`: FreeRTOS kernel plus the CMSIS-RTOS2 wrapper.
- `BSP/ST7735`: LCD BSP/driver, fonts, and logo sources. These files are included in the Keil project under the `ST7735` group.
- `External/CherryUSB-1.6.1` and `BSP/CherryUSB_port`: CherryUSB sources and the STM32 DWC2 port. Selected CDC, MSC, core, OSAL, and DWC2 sources are included in the Keil project under the `cherryusb` group.
- `MDK-ARM/`: Keil uVision project, startup assembly, debug configuration, scatter/build output folder, and the Keil-assistant `.vscode` copy.
- `moveset.bat`: copies `MDK-ARM/.vscode` into the root `.vscode` folder.

## Build And Tooling

- Primary build path is Keil MDK/uVision using `MDK-ARM/h7rec.uvprojx`.
- The target emits a hex file into `MDK-ARM/h7rec/`; treat that folder as build output.
- Active target defines are `USE_PWR_LDO_SUPPLY`, `USE_HAL_DRIVER`, and `STM32H743xx`.
- Keil include paths currently cover `Core/Inc`, HAL, CMSIS, FreeRTOS, `BSP/ST7735`, CherryUSB core/OSAL/DWC2/CDC/MSC/common under `External`, RTT under `External`, and `BSP/CherryUSB_port`.
- VS Code IntelliSense settings exist in both root `.vscode/c_cpp_properties.json` and `MDK-ARM/.vscode/c_cpp_properties.json`. The root copy appears to be generated by `moveset.bat`; avoid editing only one copy unless that divergence is intentional.
- `.gitignore` excludes Keil build products, logs, lock files, and user/workspace files. Keep source changes out of generated output such as `MDK-ARM/h7rec/`.

## Code Generation Rules

- Most files under `Core/` are CubeMX-generated. Preserve `/* USER CODE BEGIN ... */` and `/* USER CODE END ... */` regions, and place manual code inside user sections where possible.
- When changing peripheral configuration, prefer updating the `.ioc`/CubeMX source of truth and regenerating, then review the generated diff carefully.
- Do not casually reformat generated files; small behavioral edits are much easier to review when CubeMX formatting remains intact.

## Runtime And Peripheral Notes

- The project uses CMSIS-RTOS2 APIs (`cmsis_os.h`, `osThreadNew`, `osDelay`) on top of FreeRTOS.
- FreeRTOS is configured for preemption, 1 kHz tick rate, `configMAX_PRIORITIES` 56, `configMINIMAL_STACK_SIZE` 128, and `configTOTAL_HEAP_SIZE` 15360 bytes.
- The default task stack is `128 * 4` bytes. Revisit stack and heap sizes before adding heavier LCD, USB, filesystem, or protocol work.
- HAL tick uses TIM17 through `Core/Src/stm32h7xx_hal_timebase_tim.c`; do not assume SysTick owns the HAL time base.
- USB OTG FS is initialized by CubeMX as HAL PCD in `Core/Src/usb_otg.c`; CherryUSB sources are present in the Keil project, but application-level USB device descriptors/initialization are not wired into `main.c` or `freertos.c` yet.

## LCD/ST7735 Notes

- LCD-related pins are generated in `Core/Inc/main.h` and `Core/Src/gpio.c`: LCD DC/RS on PD13, reset on PD12, CS on PB12, backlight on PA5/TIM2_CH1, and key input on PB1.
- SPI2 is configured in `Core/Src/spi.c` as master, 1-line transmit, 8-bit, CPOL low, CPHA first edge, software NSS, prescaler 16, using PB13 SCK and PB15 MOSI.
- TIM2 PWM in `Core/Src/tim.c` drives backlight brightness with prescaler `480-1`, period `1000-1`, channel 1, low polarity.
- `BSP/ST7735/lcd.h` currently defines `TFT96`; switch that local BSP define deliberately if targeting a different panel variant.
- `main.c` starts TIM2 PWM before `LCD_Test()`, and the RTOS default task later writes time/tick text using `LCD_ShowString()`.

## Editing Expectations

- Keep changes scoped and compatible with Keil/ARMCC-style embedded C unless the toolchain is intentionally migrated.
- Use HAL and CMSIS-RTOS2 APIs that match the existing project style.
- Treat `BSP/ST7735`, `External/CherryUSB-1.6.1`, `External/SEGGER_RTT_V798a`, and `BSP/CherryUSB_port` as imported or ported code: fix integration issues narrowly and avoid large cosmetic rewrites.
- When touching generated CubeMX code and BSP/middleware code in the same change, separate the rationale clearly in the final response.
